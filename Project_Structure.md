# Project Structure: Agentic Vibe Fleet

This document provides a functional map of the codebase, enabling the Lead Agent (Gemini) to navigate and implement features with full architectural context.

## Core Framework (The 'Director' Layer)

| Path | Purpose |
| :--- | :--- |
| `GEMINI.md` | **Constitution**: The central nervous system and non-negotiable operating procedures. |
| `Project_Structure.md` | **Architecture Map**: This document. Functional mapping of the codebase. |
| `requirements.txt` | **Dependencies**: Python package requirements for the project. |
| `GEMINI_Getting_Started.md` | **Onboarding**: Auto-updated guide on using Gemini Code Assist features. |
| `PATTERNS.md` | **Pattern Registry**: Living document for established engineering patterns and design decisions. |
| `scripts/` | **Agentic Skills**: Maintenance and hygiene scripts accessible to agents. |
| `bruno/` | **API Validation**: Bruno collections and documentation for contract testing. |
| `bootstrap_prompts/` | **Plan Archive**: Systematic prompts generated from user intent to start new sessions. |
| `terraform/` | **Infrastructure-as-Code** (Slice 10): Terraform codifying the GCP platform — enabled APIs, Firestore DB, runtime-SA IAM, public-invoker binding, daily Cloud Scheduler job (Cloud Run service read as a data source; deployed by gcloud). `validate`+`plan` clean; adopt live infra via `import.sh`. ~$0/mo. |

## Application Layer

| Path | Purpose |
| :--- | :--- |
| `src/main.py` | **API**: FastAPI app. `GET /health` + `GET /api/v1/offers/cheapest` (Slice 1); watch routes are owner-scoped via `_owned_watch` (Slice 8 — private per user, admin sees all). |
| `src/config.py` | **Settings**: Env-driven config (provider choice, Amadeus creds, timeouts). |
| `src/obs.py` | **Observability** (Slice 11): dependency-free structured logging — JSON on Cloud Run (`jsonPayload.*`), text locally; `configure_logging()` + `event()` + `get_logger()`; `LOG_LEVEL`-controlled. Query guide in `docs/LOGGING.md`. |
| `src/models.py` | **Domain**: Provider-agnostic `Offer` model (the time-series record). |
| `src/providers/base.py` | **Seam**: `FareProvider` ABC + `ProviderError`. The data-source contract. |
| `src/providers/mock.py` | **Provider**: Deterministic, network-free mock (default; test-safe, zero quota). |
| `src/providers/travelpayouts.py` | **Provider**: Travelpayouts/Aviasales `/v1/prices/cheap` (live, free; cached price-trend data). The active live source after Amadeus shut down. |
| `src/providers/amadeus.py` | **Provider**: Amadeus Self-Service impl (DEPRECATED — Self-Service decommissioned 2026-07-17; kept for reference). |
| `src/providers/factory.py` | **Selector**: Resolves the active provider from `FARE_PROVIDER`. |
| `src/store/base.py` | **Seam**: `WatchRepository` ABC + `WatchNotFoundError` + `build_snapshot`. The persistence contract; watches carry `owner_email` and `list_watches` can scope to one owner (Slice 8). |
| `src/store/sqlite_store.py` | **Store**: Default SQLite impl (stdlib, durable, hermetic; FK cascade). |
| `src/store/firestore_store.py` | **Store**: Cloud Firestore impl (`WATCH_STORE=firestore`); used in prod. |
| `src/store/factory.py` | **Selector**: Resolves the active store from `WATCH_STORE` (sqlite \| firestore). |
| `src/auth.py` | **Auth** (Slice 6): Google ID-token verification (+ dev-mode passwordless login), signed-session helpers, and the `require_user` / `require_admin` FastAPI gate dependencies. |
| `src/users/base.py` | **Seam** (Slice 6): `AllowedUserRepository` ABC — the access-allowlist persistence contract. |
| `src/users/sqlite_store.py` | **Store** (Slice 6): default SQLite allowlist (stdlib; idempotent upsert). |
| `src/users/firestore_store.py` | **Store** (Slice 6): Cloud Firestore allowlist (`allowed_users/{email}`); used in prod. |
| `src/users/factory.py` | **Selector** (Slice 6): resolves the allowlist store from `WATCH_STORE` (shares the watch backend). |
| `src/poll.py` | **Poll engine**: Quota-capped `poll_active_watches` + `PollSummary`; prices each watch's trip (Slice 7) then runs detection + alerting after each recorded snapshot (Slice 3a/4). |
| `src/trip.py` | **Trip pricing** (Slice 7/9): `price_watch_trip` resolves a watch to its legs (`Watch.resolved_legs`: one-way=1, round-trip=2, multi-city=N), window-searches each leg via the `FareProvider` seam (cheapest date in a flex window), and sums them into a `TripQuote`; provider-agnostic. |
| `src/signal.py` | **Signal detection**: pure `detect_reopening` (7-day moving avg, >15% rising-edge drop) + `Signal`/`SignalResult` (Slice 4). |
| `src/alerts/base.py` | **Seam**: `AlertChannel` ABC + `AlertError` + shared `render_text`. The notification contract. |
| `src/alerts/log_channel.py` | **Channel**: default `LogAlertChannel` — logs the alert (zero-cost, prod-safe). |
| `src/alerts/webhook_channel.py` | **Channel**: `WebhookAlertChannel` — POSTs JSON to a Slack/Discord/generic incoming webhook (httpx). |
| `src/alerts/factory.py` | **Selector**: resolves the active channel from `ALERT_CHANNEL` (log \| webhook). |
| `src/web/dashboard.html` | **Dashboard** (Slice 5/6/7): self-contained single-page UI (vanilla JS + inline SVG sparklines) served at `GET /` (auth-gated; redirects to `/login` if signed out); add-watch form supports one-way / round-trip + per-leg date flexibility; cards show the trip total, per-leg breakdown, "Bucket reopened" badge, signed-in user + sign-out + an Admin link for the owner. |
| `src/web/login.html` | **Login** (Slice 6): sign-in page at `GET /login`; renders "Sign in with Google" (GIS) in prod or a passwordless email form in `AUTH_MODE=dev`. |
| `src/web/admin.html` | **Admin console** (Slice 6): owner-only allowlist management at `GET /admin` (add / list / revoke invited Gmail accounts). |
| `Dockerfile` | **Container**: Cloud Run image (python:3.13-slim, uvicorn on `$PORT`). |
| `firebase.json` + `.firebaserc` | **Hosting** (Slice 12): Firebase Hosting serves the app at `https://cheapsawari.web.app`, rewriting all paths to the Cloud Run service `cheapsawari-api` (the `public/` dir is intentionally empty). The Cloud Run URL still works directly. |
| `.dockerignore` | Keeps the build context to `src/` + `requirements.txt`. |
| `bruno/cheapsawari/` | **API Validation**: Bruno collection — offers + watches + poll + signal + dashboard (green on both sqlite and Firestore). |
| `docs/DEPLOYMENT.md` | **Runbook**: Live cloud resources, env, redeploy, cost review. |
| `docs/architecture_overview.html` | **Visual Guide**: A 1-page HTML overview of the Agentic Vibe Fleet framework. (Excluded from `verify_structure.py` checks) |
| `Function_Mapping.md` | **Traceability Map**: Correlates frontend components with backend API functions. |

> **Roadmap (rolling MVP):** Slice 1 — fare-fetch seam ✅ · Slice 2 — watches + persistence ✅ · Slice 3a — quota-capped poll engine ✅ · Slice 3b — cloud wiring (Firestore + Cloud Run + Cloud Scheduler) ✅ LIVE · Slice 4 — signal detection + single-channel alert ✅ LIVE · Slice 5 — minimal dashboard ✅ · Slice 6 — Google auth gate + admin allowlist ✅ · Slice 7 — round-trip + date-range flexibility ✅ · Slice 8 — per-user watch ownership ✅ · Slice 9 — multi-city trips ✅ (N user-defined legs; also delivers transfer-airport & port-of-entry choice as explicit legs, on the current $0 route-level data). **Next (planned):** terraform infra-as-code (AVF Pattern 5 tech-debt, before any release tag); optional itinerary provider only if true through-fares are later needed. _(LIVE on real Travelpayouts fares since 2026-06-24.)_
